## Configuration

### Step 1: Prepare Your UC Remote Device

**IMPORTANT**: Your UC Remote must be powered on and connected to the same network as your Android device.

#### Verify Remote Connection:
1. Ensure Remote is connected to WiFi network
2. Note the IP address from Remote's settings
3. Ensure Remote firmware is up to date
4. Verify Remote is accessible on network

#### Network Setup:
- **Same Network**: App and Remote must be on same WiFi
- **Static IP**: Recommended via DHCP reservation
- **Firewall**: No firewall blocking required ports
- **Local Access**: Cloud connection not supported

### Step 2: Connect to Remote

#### **Initial Setup:**

1. Open UC Remote Android app
2. Tap **"Plus Symbol"** To add new remote
3. Enter Remote IP
4. Enter Remote PIN
5. Click **"Test Connection"**
6. Once connection test passed, tap **"Save"** 
7. Repeat for multiple remotes

**Connection Test:**
- App verifies Remote connectivity
- Authentication tested
- Profile loaded automatically
- Entities synchronized

### Step 3: Select Profile

After successful connection:

1. Tap profile selector (top-right corner)
2. Select active profile from list
3. App synchronizes profile data
4. Entities and pages loaded

- **Activities** - Start/stop activities, view running activities
- **Entities** - Browse all entities by type
- **Profile Pages** - Navigate custom profile pages
- **Integrations** - View and manage integrations

1. **Tap profile selector** (top-right)

3. **Wait for sync** (~500ms)

1. **Tap menu** (top-left)

3. **Options available:**

## Troubleshooting

### Cannot Connect to Remote

**Symptoms:**
- Connection timeout
- "Remote not found" error
- Authentication failed

**Solutions:**
1. Verify Remote is powered on
2. Check both devices on same WiFi network
3. Verify PIN code is correct
4. Restart Remote device
5. Check firewall settings

### Entities Not Loading

**Symptoms:**
- Empty entity list
- "Loading..." never completes
- Entities show as "unavailable"

**Solutions:**
1. Pull down to refresh entity list
2. Switch profiles and switch back
3. Reconnect to Remote
4. Check Remote has integrations configured
5. Verify WebSocket connection (check logs)

### Real-Time Updates Not Working

**Symptoms:**
- Entity states don't update
- Changes on Remote not reflected in app
- Delayed state updates

**Solutions:**
1. Check WebSocket connection in logs
2. Reconnect to Remote
3. Restart app
4. Check network stability
5. Verify Remote firmware version

### Profile Switching Fails

**Symptoms:**
- Profile switch timeout
- Entities disappear
- App freezes during switch

**Solutions:**
1. Wait 5 seconds and try again
2. Reconnect to Remote
3. Restart app
4. Check Remote profile exists
5. Verify network connection

### App Crashes or Freezes

**Symptoms:**
- App force closes
- UI becomes unresponsive
- Screen goes black

**Solutions:**
1. Force stop app and restart
2. Clear app cache (Settings ‚Üí Apps)
3. Update to latest version
4. Report crash logs to developer
5. Reinstall app if persistent

## Technical Details

### Architecture

**Pattern**: MVVM (Model-View-ViewModel)
**Language**: Kotlin (100% native Android)
**UI Framework**: Material Design 3
**Navigation**: Android Navigation Component with Fragment-based navigation
**State Management**: StateFlow for reactive UI updates
**Networking**: OkHttp + Retrofit for REST, OkHttp WebSocket for real-time
**Concurrency**: Kotlin Coroutines + Dispatchers
**View Binding**: ViewBinding for type-safe view access
**Persistence**: SharedPreferences + JSON serialization
